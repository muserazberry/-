# 경기도 조례 제·개정 분석 자동화 시스템

정부 정책·국회 입법·도민 민원 등을 참고해 **경기도에 제·개정이 필요한 조례를 추천**하는 시스템입니다.

> 핵심 구조: **신호(signal) → 경기도 조례 대조 → 미반영 추천**.
> 출처(국회 법안 / 정부·경기도 정책·보도자료 …)만 갈아끼우면 같은 엔진이 처리합니다.

## 전체 구상 (6단계)

1. 국가법령정보센터에서 경기도·경기도교육청 조례 검토
2. 국회 통과/계류 법률(안)과 심사보고서 검색
3. **통과 법안 중 경기도 미반영 조례안 검토**  ← 출처 `assembly`
4. **정책·공약·부처 보고서 기반 신규 조례 필요성 파악**  ← 출처 `policy` (RSS)
5. **보도자료로 입법·정책 추진 동향 파악**  ← 출처 `policy` (RSS)
6. 제·개정 필요 조례(안) 추천  ← 추천 엔진이 ③④⑤ 결과를 동일 형식으로 산출

> **선제 대응**: 통과 전 단계인 **법제처 입법예고**(출처 `lawmaking`)도 같은 엔진으로 분석해, 법안이 확정되기 전에 영향받을 경기도 조례를 미리 파악합니다.

③을 먼저 끝까지 검증한 뒤, 코어를 **출처 무관 추천 엔진**으로 일반화해 ④⑤(정책/보도자료 RSS)를 같은 흐름으로 붙였습니다. 민원(국민신문고)은 공개 API가 마땅찮아 다음 후보로 둡니다.

## 구조

```
app/
  config.py              환경설정·상수 (키, RSS 피드)
  sources.py             출처 어댑터 → 공통 signal 로 변환 (assembly/policy)
  clients/
    assembly_client.py   열린국회정보 OPEN API (법률안)
    law_client.py        국가법령정보센터 OPEN API (자치법규 + 법령 본문)
    lawmaking_client.py  정부입법지원센터 입법예고 (통과 전, 선제 대응)
    council_client.py    경기도의회 입법예고 게시판 HTML 파싱 (조례안 동향)
    epeople_client.py    국민신문고 민원 질의응답 / 국민생각함 국민제안 HTML 파싱 (참고)
    seminar_client.py    국회도서관 세미나 일정 (열린국회정보 API, 정책 동향·참고)
    budget_client.py     세출예산 CSV → 신규/대규모/증액 사업 판정
    rss_client.py        범용 RSS/Atom 수집 (정책·보도자료)
  analysis/
    keywords.py          제목 → 핵심 키워드 (+ 동의어·약어 통일)
    matching.py          제목 vs 조례명 관련도 (연계·핵심어 일치 + 바이그램)
    delegation.py        법령 본문 → 조례 위임조항("조례로 정한다") 탐지
    recommender.py       신호 → 조례 대조 추천 엔진 (출처 무관)
    gap_finder.py        오케스트레이터 (출처 선택 → 추천)
  api/main.py            FastAPI 백엔드 + 대시보드 서빙
web/index.html           단일 페이지 대시보드 (출처 선택)
```

## 설치 및 실행

```powershell
cd gyeonggi-ordinance-system
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt

copy .env.example .env   # 키 입력
uvicorn app.api.main:app --reload
```

> **가장 쉬운 실행:** 탐색기에서 `start.bat` 더블클릭. 가상환경 활성화 없이 서버를 켜고 브라우저를 자동으로 엽니다. (`uvicorn` 명령을 못 찾는 문제를 피하려고 `.venv`의 파이썬을 직접 호출합니다.)

브라우저에서 http://127.0.0.1:8000 접속 → [분석 실행].

핵심 로직(키워드·매칭·추천 엔진) 회귀 테스트는 네트워크 없이 표준 라이브러리로 돌아갑니다:

```powershell
python -m unittest discover -s tests
```

## 키 적용 후 확인이 필요한 값

웹 문서가 막혀 있어 아래 두 값은 **본인 대시보드에서 1회 확인**을 권장합니다. 모두 `.env`로 교체 가능하도록 설계했습니다.

- `ASSEMBLY_BILL_SERVICE`: 구독한 의안 서비스 ID가 `TVBPMBILL11`과 다르면 교체.
- `POLICY_FEEDS`: 정책브리핑·경기도 등 **실제 RSS 피드 URL**을 각 사이트에서 확인 후 입력. (파서는 RSS 2.0/Atom 모두 검증 완료, URL만 사이트마다 다름)
- 경기도 조례 필터: 키워드 검색 후 `지자체기관명`에 "경기도"가 포함된 항목만 추립니다(org 코드 하드코딩 회피).
- 입법예고(`lawmaking`): 엔드포인트는 `lawmaking.go.kr/rest/ogLmPp`(법제처 공식, XML)로 확인했고 응답 태그(lsNm/stYd/edYd/asndDptNm)·표기 변형을 흡수하도록 파싱합니다. **목록 페이징 파라미터만 본인 키로 1회 응답 확인**을 권장합니다(인증·HTTP 오류는 502로 표면화).

## 동작 방식

1. **신호 수집** (`sources.py`) — 출처 선택
   - `assembly`: 열린국회정보에서 법률안 조회 → **가결** 건만 선별
   - `lawmaking`: 정부입법지원센터 **입법예고**(통과 전) 목록 — 확정 전 선제 대응
   - `council`: **경기도의회 입법예고**(조례안) — 도의회가 직접 발의·예고 중인 조례안. 제목이 곧 조례안명이라 기존 경기도 조례와 매칭이 정확함(일부개정 → 기존 조례 / 제정 → 신규)
   - `budget`: 경기도 **세출예산 CSV** → **신규·대규모·증액 사업**만 추려 신호화. (`예산구분` 메타로 신규/대규모/증액 표시) 단, **사업명(세부사업명) ↔ 조례명 매칭은 표기 특성상 불확실**하므로 예산 신호는 **참고용**으로만 분류합니다 — 확정 추천(미반영)·고신뢰 갭으로 세지 않고, 관련 가능성 있는 조례 후보만 함께 보여줍니다(통계의 `참고` 건수).
   - `minwon`/`idea`: **국민신문고 민원 질의응답** / **국민생각함 국민제안**(전국 단위) — 도민·국민 정책 수요를 참고. 전체 수집 후 **소관기관**을 메타 태그로 분류. 제목↔조례명 매칭이 부정확해 **참고용**으로만 노출(예산과 동일)
   - `seminar`: **국회도서관 세미나 일정**(열린국회정보 API, `ASSEMBLY_KEY` 재사용) — 정책 동향 신호. 세미나 제목은 조례명이 아니라 정책 주제라 매칭이 부정확해 **참고용**으로만 노출
   - `policy`: `POLICY_FEEDS` RSS에서 정부·경기도 정책/보도자료 수집
2. 각 신호 제목에서 조사·일반어를 제거하고 **핵심어 후보 2개** 추출 (`keywords.py`)
   - **동의어·약어 통일**: `AI`=`인공지능`, `저출산`=`저출생` 처럼 표기가 달라 매칭을 놓치는 경우를 줄입니다. 검색 시 동의어를 함께 질의(재현율↑)하고, 비교 시 대표어로 통일합니다.
3. 국가법령정보센터에서 핵심어들로 **경기도** 자치법규를 조회해 합집합 구성
4. 조례별 관련도 판정 (`matching.py`) — 3단계
   - **이미 반영됨**: 공식 연계됐거나 신호 주제어가 조례명에 포함(핵심어 일치)
   - **유사·검토**: 일부 토큰만 겹침
   - **미반영(추천)**: 관련 조례 없음
   - 보조로 글자 바이그램 유사도(0~1)를 매겨 같은 등급 안에서 정렬
5. **법령 본문 분석** (`assembly`·`lawmaking` 등 법령안 출처에서 "위임조항 분석" ON일 때)
   - 법령안명에서 대상 법령명을 추정합니다.
   - **공식 연계 조회** (`law_client`, target `lnkLs`): 법제처가 직접 연결한 **경기도 연계 자치법규**를 가져옵니다. 연계 조례가 있으면 키워드로 못 찾아도 권위 있는 **이미 반영(🔗연계)**으로 판정합니다.
   - **위임조항 분석** (`delegation.py`): 법령 본문에 "…는 조례로 정한다" 같은 **조례 위임조항**이 있으면, 경기도 미반영 시 **고신뢰(위임○) 추천**으로 승격합니다. 위임조항이 없는 법(형법 등 전국 일률 적용)은 후순위로 내려 오탐을 줄입니다.
6. 미반영·위임조항 우선 정렬해 대시보드 표로 표시 (원문 링크·유사도·연계/핵심어 일치·위임 근거 스니펫 표시)

## 새 출처(단계) 추가하기

`sources.py`에 `signal` 리스트를 돌려주는 함수를 만들고 `SOURCES`에 등록하면 끝입니다.
signal 형식: `{"source", "title", "summary", "link", "date", "meta"}`. 추천 엔진·대시보드는 수정 불필요.

## 한계와 다음 단계

- **법령 본문 분석(위임조항+공식 연계)**은 의안 1건마다 API를 호출하므로 표본이 크면 느립니다. 대시보드의 "위임조항 분석" 체크박스(API `with_delegation`)로 끌 수 있고, 같은 법령은 캐시합니다.
- **공식 연계**(target `lnkLs`)는 지자체기관명 필드가 없어 자치법규명에 "경기도" 포함 여부로 경기도 항목을 추립니다. 연계 조회가 비거나 실패하면 키워드 매칭으로 자동 폴백합니다(fail-safe).
- `keywords.py`의 동의어 그룹(`_SYNONYM_GROUPS`)·일반어 목록(`_GENERIC`), `delegation.py`의 위임 표현 패턴(`_PATTERNS`), `matching.py`의 일치 규칙은 실제 데이터를 보고 손쉽게 조정할 수 있습니다.
- **예산(`budget`)**은 사업명+금액이 들어있는 **세출예산 CSV**를 입력으로 씁니다(업무보고 PDF/HWP 본문 파싱은 향후 과제). '대규모'는 단위를 모르므로 절대 금액 임계 대신 **금액 상위 N개**로 판정합니다. 컬럼명은 흔한 한글 표기를 자동 인식하나, 특이한 서식이면 헤더 확인이 필요합니다. 예산 신호는 **참고용**으로 표시되어(사업명↔조례명 매칭 불확실) 미반영 추천 건수에는 포함되지 않습니다.
- **경기도의회 입법예고(`council`)**는 RSS·공공데이터 API가 없어 `ggc.go.kr` 게시판 **HTML을 직접 파싱**합니다. 인증키는 필요 없지만, 사이트 마크업이 바뀌면 `council_client.py`의 정규식(`_LINK` 등) 조정이 필요합니다. 회의록은 변수가 많아 제외하고, 도의회 입법예고로 도 차원 입법 동향을 더 정확히 포착합니다.
- **국민신문고 민원 질의응답·국민생각함 제안(`minwon`/`idea`)**은 `epeople.go.kr` 목록 **HTML을 직접 파싱**합니다(인증키 불필요). 전국 단위라 노이즈가 있어 **참고용**으로만 노출하고 소관기관을 태그로 분류합니다. 상세 페이지가 JS 함수로 열려 원문 직링크는 제공하지 않습니다. 사이트 마크업이 바뀌면 `epeople_client.py` 정규식 조정이 필요합니다.
- **국회도서관 세미나(`seminar`)**는 열린국회정보 OPEN API를 사용합니다(`ASSEMBLY_KEY` 재사용). 공식 서비스는 국회도서관이 제공하는 **"국회의원 세미나 일정"**(무료·실시간)입니다 → [서비스 페이지](https://open.assembly.go.kr/portal/data/service/selectServicePage.do?infId=O67B1I001080WL10254). 법률안 서비스처럼 **본인 대시보드에서 'OPEN API 신청' 후 영문 서비스명**을 확인해 `ASSEMBLY_SEMINAR_SERVICE`에 입력하세요(서비스명·인증키는 로그인 시에만 노출). 서비스별로 응답 컬럼명 표기가 달라 흔한 후보(`TITLE`/`DATE`/`HOST`/`PLACE` 등)를 자동 인식하며, 필드명이 다르면 `seminar_client.py`의 후보 목록만 조정하면 됩니다.
